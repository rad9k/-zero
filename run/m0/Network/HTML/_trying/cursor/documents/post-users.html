POST /api/users

Tworzy nowego użytkownika.

Body

{
  "name": "Jan Kowalski",
  "email": "jan@example.com",
  "password": "securePassword123",
  "role": "user"
}

Parametry

  • name (wymagany): Imię i nazwisko użytkownika
  • email (wymagany): Adres email (unikalny)
  • password (wymagany): Hasło (min. 8 znaków)
  • role (opcjonalny): Rola użytkownika (user, admin)

Odpowiedź

{
  "id": 123,
  "name": "Jan Kowalski",
  "email": "jan@example.com",
  "role": "user",
  "createdAt": "2024-01-01T00:00:00Z",
  "updatedAt": "2024-01-01T00:00:00Z"
}

Kody odpowiedzi

  • 201 - Użytkownik utworzony
  • 400 - Błędne dane
  • 409 - Email już istnieje
  • 401 - Brak autoryzacji
  • 403 - Brak uprawnień

Walidacja

  • Email musi być unikalny
  • Hasło musi mieć minimum 8 znaków
  • Nazwa nie może być pusta
  • Email musi być poprawnym formatem
