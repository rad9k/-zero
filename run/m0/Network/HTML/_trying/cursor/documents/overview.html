Przegląd systemu

System składa się z kilku głównych komponentów, które współpracują ze sobą.

Architektura

System wykorzystuje architekturę modułową z jasno zdefiniowanymi interfejsami.

Główne komponenty

  • Serwer HTTP - obsługuje żądania HTTP
  • Router - kieruje żądania do odpowiednich kontrolerów
  • Kontrolery - obsługują logikę biznesową
  • Modele - reprezentują dane w systemie
  • Baza danych - przechowuje dane aplikacji
  • Cache - przyspiesza dostęp do często używanych danych

Przepływ danych

  1. Żądanie HTTP dociera do serwera
  2. Router kieruje żądanie do odpowiedniego kontrolera
  3. Kontroler przetwarza żądanie i używa modeli
  4. Model pobiera/zapisuje dane z bazy danych
  5. Kontroler zwraca odpowiedź HTTP

Technologie

  • Node.js - środowisko uruchomieniowe
  • Express.js - framework web
  • SQLite/PostgreSQL - baza danych
  • Redis - cache (opcjonalnie)
  • JWT - autoryzacja
