Architektura

Architektura systemu opiera się na wzorcu MVC (Model-View-Controller).

Warstwy systemu

  • Warstwa prezentacji: Interfejs użytkownika
  • Warstwa logiki biznesowej: Kontrolery i serwisy
  • Warstwa danych: Modele i repozytoria

Wzorzec MVC

Model

Reprezentuje dane i logikę biznesową. Modele definiują strukturę danych i metody do ich manipulacji.

View

Odpowiada za prezentację danych użytkownikowi. W naszym przypadku to interfejs webowy.

Controller

Obsługuje żądania użytkownika, koordynuje między modelem a widokiem.

Modułowość

System jest podzielony na moduły:

  • Core - podstawowe funkcjonalności
  • Auth - autoryzacja i uwierzytelnianie
  • API - interfejs programistyczny
  • Admin - panel administracyjny
  • Plugins - rozszerzenia systemu

Komunikacja między modułami

Moduły komunikują się przez:

  • Interfejsy (interfaces)
  • Eventy (events)
  • Dependency injection
