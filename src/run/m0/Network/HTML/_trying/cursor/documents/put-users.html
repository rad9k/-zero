PUT /api/users/:id

Aktualizuje istniejącego użytkownika.

Parametry ścieżki

  • id: ID użytkownika do aktualizacji

Body

{
  "name": "Jan Kowalski (zaktualizowany)",
  "email": "jan.nowy@example.com",
  "role": "admin"
}

Parametry

  • name (opcjonalny): Nowe imię i nazwisko
  • email (opcjonalny): Nowy adres email
  • role (opcjonalny): Nowa rola użytkownika
  • password (opcjonalny): Nowe hasło

Odpowiedź

{
  "id": 123,
  "name": "Jan Kowalski (zaktualizowany)",
  "email": "jan.nowy@example.com",
  "role": "admin",
  "createdAt": "2024-01-01T00:00:00Z",
  "updatedAt": "2024-01-02T00:00:00Z"
}

Kody odpowiedzi

  • 200 - Użytkownik zaktualizowany
  • 400 - Błędne dane
  • 404 - Użytkownik nie znaleziony
  • 409 - Email już istnieje
  • 401 - Brak autoryzacji
  • 403 - Brak uprawnień

Uwagi

  • Wszystkie pola są opcjonalne
  • Tylko podane pola zostaną zaktualizowane
  • Email musi być unikalny (jeśli jest zmieniany)
  • Hasło jest hashowane przed zapisem
