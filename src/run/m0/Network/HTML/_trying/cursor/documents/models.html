Modele danych

Modele danych definiują strukturę informacji w systemie.

Wszystkie modele są walidowane przed zapisem do bazy danych.

Struktura modelu

Każdy model zawiera:

  • Definicję pól i ich typów
  • Reguły walidacji
  • Relacje z innymi modelami
  • Metody do manipulacji danymi

Dostępne modele

  • User - użytkownicy systemu
  • Product - produkty w systemie
  • Document - dokumenty
  • Category - kategorie
  • Order - zamówienia

Walidacja

Wszystkie modele są automatycznie walidowane:

  • Wymagane pola
  • Format danych
  • Unikalność wartości
  • Relacje między modelami

Przykład definicji modelu

class User {
  id: number;
  name: string;
  email: string;
  role: string;
  createdAt: Date;
  updatedAt: Date;
  
  validate() {
    // Logika walidacji
  }
}
